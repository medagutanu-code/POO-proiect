# POO-proiect
Programul modelează funcționarea unei agenții imobiliare, în care se gestionează relația dintre proprietăți, clienți și ofertele generate între aceștia. Există trei clase principale: Imobil, Client și Oferta. Clasa Imobil descrie o proprietate disponibilă, având informații precum locația, prețul, suprafața și existența unui balcon. Clasa Client reține preferințele unui cumpărător, inclusiv bugetul, locația dorită și caracteristicile minime acceptate. Clasa Oferta face legătura între un client și un imobil și calculează un scor care reflectă cât de bine se potrivește proprietatea cerințelor clientului.

În funcția main, sunt create mai multe obiecte de tip Imobil și Client, care sunt apoi stocate în vectori. Programul parcurge fiecare client și, pentru fiecare dintre ei, analizează toate imobilele disponibile. Pentru fiecare pereche client–imobil, se creează o ofertă și se verifică dacă locația dorită de client coincide cu cea a imobilului. Dacă această condiție este îndeplinită, se calculează un scor bazat pe diferența de buget, suprafață și existența balconului.

Pe parcurs, se reține oferta cu scorul cel mai mare pentru fiecare client. La finalul analizei, programul afișează datele clientului și cea mai bună ofertă găsită. Dacă scorul depășește un prag stabilit, oferta este marcată ca fiind aprobată. În caz contrar, se indică faptul că nu există o ofertă potrivită.
